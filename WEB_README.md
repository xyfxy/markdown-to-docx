# Markdown to DOCX Web应用

## 🌟 功能特点

这是一个现代化的Web界面，为Markdown转DOCX工具提供了友好的用户体验：

### 📁 文件上传模式
- **拖拽上传**: 直接拖拽.md或.txt文件到上传区域
- **点击选择**: 点击按钮选择本地文件
- **文件验证**: 自动检查文件格式和大小（最大16MB）
- **实时预览**: 显示选中文件的信息

### ✏️ 在线编辑模式
- **实时编辑**: 在浏览器中直接编写Markdown
- **字符计数**: 实时显示字符数量
- **语法提示**: 内置Markdown语法参考
- **即时转换**: 无需保存文件即可转换

### ⚙️ 自定义选项
- **字体大小**: 自定义正文字体大小（8-72pt）
- **行间距**: 自定义行间距（10-50pt）
- **格式优化**: 专为中文文档优化

### 🎨 界面特点
- **响应式设计**: 支持桌面和移动设备
- **现代UI**: 使用Bootstrap 5和自定义样式
- **动画效果**: 流畅的交互动画
- **深色模式**: 自动适应系统主题

## 🚀 快速开始

### 1. 启动Web服务

```bash
# 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 启动Web应用
python run_web.py
```

### 2. 访问应用

浏览器会自动打开 `http://localhost:5000`

如果没有自动打开，请手动访问该地址。

### 3. 使用应用

#### 方式一：文件上传
1. 点击"文件上传"标签页
2. 拖拽或选择.md/.txt文件
3. 设置字体大小和行间距（可选）
4. 点击"开始转换"
5. 下载生成的DOCX文件

#### 方式二：在线编辑
1. 点击"在线编辑"标签页
2. 在文本框中输入Markdown内容
3. 设置格式选项（可选）
4. 点击"预览转换"
5. 下载生成的DOCX文件

## 📋 支持的Markdown语法

- **标题**: `# H1`, `## H2`, `### H3`, `#### H4`, `##### H5`
- **段落**: 空行分隔的文本块
- **列表**: 
  - 无序列表: `- 项目` 或 `* 项目`
  - 有序列表: `1. 项目`
  - 嵌套列表: 使用缩进
- **强调**: 
  - 粗体: `**文本**`
  - 斜体: `*文本*`
- **中英文混合**: 自动处理字体

## 🎯 输出特点

生成的DOCX文档具有以下特点：

- **中文优化**: 使用仿宋、黑体、楷体等中文字体
- **标准格式**: 符合公文写作规范
- **智能缩进**: 段落首行缩进、列表层级缩进
- **收件人识别**: 自动识别以冒号结尾的收件人行
- **页面设置**: A4纸张，标准页边距

## 🔧 技术架构

### 后端
- **Flask**: Python Web框架
- **Click**: 命令行接口集成
- **python-docx**: DOCX文件生成
- **markdown**: Markdown解析

### 前端
- **Bootstrap 5**: 响应式UI框架
- **Bootstrap Icons**: 图标库
- **原生JavaScript**: 交互逻辑
- **CSS3**: 动画和样式

### 文件结构
```
├── app.py              # Flask应用主文件
├── run_web.py          # Web应用启动脚本
├── templates/
│   └── index.html      # 主页模板
├── static/
│   └── style.css       # 自定义样式
├── uploads/            # 文件上传目录（自动创建）
├── main.py             # 原CLI工具
├── md_parser.py        # Markdown解析器
└── docx_generator.py   # DOCX生成器
```

## 🛠️ 开发说明

### 添加新功能
1. 在`app.py`中添加新的路由
2. 在`templates/index.html`中添加UI元素
3. 在JavaScript中添加交互逻辑

### 自定义样式
编辑`static/style.css`文件来修改界面样式。

### 调试模式
在`app.py`中设置`debug=True`来启用调试模式。

## 🔒 安全注意事项

- 文件上传大小限制为16MB
- 只允许.md和.txt文件上传
- 上传文件名会被安全处理
- 临时文件会自动清理

## 📞 故障排除

### 常见问题

1. **端口被占用**
   ```bash
   # 更改端口（在app.py中修改）
   app.run(port=5001)
   ```

2. **文件上传失败**
   - 检查文件格式（只支持.md和.txt）
   - 检查文件大小（最大16MB）
   - 确保有足够的磁盘空间

3. **转换失败**
   - 检查Markdown语法是否正确
   - 查看浏览器控制台的错误信息
   - 检查服务器日志

### 日志查看
Web应用会在控制台输出详细的日志信息，包括：
- 请求处理状态
- 文件上传信息
- 转换过程日志
- 错误堆栈信息
